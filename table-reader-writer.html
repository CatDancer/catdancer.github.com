table-reader-writer

Reader / writer for Arc tables

This patch to Arc adds a reader and writer for Arc tables so that they are printed out in a way that they can be read back in.

Although this patch implements a particular syntax for table literals (a simple unstructed list of keys and values within curly braces), my main goal was implementing the reader / writer part; the patch can easily be hacked to some other syntax if you prefer another.

 arc> (obj "Boston" 'bos "San Francisco" 'sfo "Paris" 'cdg)
 {"San Francisco" sfo "Paris" cdg "Boston" bos}
 arc> {"San Francisco" sfo "Paris" cdg "Boston" bos}
 {"San Francisco" sfo "Paris" cdg "Boston" bos}

Unlike tables read using MzScheme’s #hash((...)) syntax, tables read in by this reader can be modified.

Contains

This patch is built upon the following patches:

  • scheme-values Allows Scheme values to pass through the Arc compiler.
  • list-writer A writer for Arc lists.

Apply This Hack to Your Arc

By using git or patch, you can incorporate this patch into your version of Arc.

With git

 $ git pull git://github.com/CatDancer/arc.git tag arc2.table-reader-writer0

For example,

 $ mkdir arc
 $ cd arc
 $ git init
 $ git pull git://github.com/CatDancer/arc.git tag arc2.table-reader-writer0

With patch

To apply this patch to your copy of arc using the patch command, download arc2.table-reader-writer0.patch into your Arc directory, and at the Unix command line inside the Arc directory, type:

 patch <arc2.table-reader-writer0.patch

“patch” is a standard utility that comes with Unix. (If you’re running Windows, iirc patch comes with cygwin).

For example,

 $ wget http://ycombinator.com/arc/arc2.tar
 [... downloading ...]
 $ tar xf arc2.tar
 $ cd arc2
 $ patch <arc2.table-reader-writer0.patch
 patching [...]
 $
