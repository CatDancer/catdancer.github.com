Send output going to stdout to stderr instead

This is a little macro that I find useful. It takes output that normally would go to stdout and sends it to stderr instead. (I call it toerr by analogy with tostring, which takes output going to stdout and puts it in a string).

When used inside of a defop, it causes output to print on the Arc REPL instead of being sent to the user’s browser.

 (defop foo req
   (toerr:ppr ...)

Apply This Hack to Your Arc

By using git or patch, you can incorporate this patch into your version of Arc.

With git

 $ git pull git://github.com/CatDancer/arc.git tag arc2.toerr0

For example,

 $ mkdir arc
 $ cd arc
 $ git init
 $ git pull git://github.com/CatDancer/arc.git tag arc2.toerr0

With patch

To apply this patch to your copy of arc using the patch command, download arc2.toerr0.patch into your Arc directory, and at the Unix command line inside the Arc directory, type:

 patch <arc2.toerr0.patch

“patch” is a standard utility that comes with Unix. (If you’re running Windows, iirc patch comes with cygwin).

For example,

 $ wget http://ycombinator.com/arc/arc2.tar
 [... downloading ...]
 $ tar xf arc2.tar
 $ cd arc2
 $ patch <arc2.toerr0.patch
 patching [...]
 $
